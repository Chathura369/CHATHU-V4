## 🚀 Quick Start

### 📋 Prerequisites

- **Node.js**: 18.x or higher
- **Git**: Installed for repository management
- **FFmpeg**: Required for media processing

### ⚙️ Installation & Launch

```bash
# Clone the repository
git clone https://github.com/Chathura369/CHATHUMD-V3.git
cd CHATHUMD-V3

# Install dependencies
npm install

# Start the bot
npm start
```

### 🖥️ Accessing the Dashboard

Once the bot starts, navigate to:
👉 **`http://localhost:5000`**

Use the dashboard to scan the QR code or link via Phone Number pairing.

## ☁ Deploying to Free Hosts

The repo ships with first-class config for the most common free WhatsApp-bot
hosts: Railway, Render, Fly.io, and any Docker-friendly platform.

### Common environment variables

| Var | Purpose | Notes |
| --- | --- | --- |
| `PORT` | Dashboard port | Auto-injected by Railway / Render / Fly. Leave unset there. |
| `HOST` | Bind interface | Defaults to `0.0.0.0` (correct for containers). |
| `DATA_DIR` | Persistent state root | Point at the mounted volume so `db.json`, WhatsApp creds, downloads survive restarts. |
| `ADMIN_USER` / `ADMIN_PASS` | Dashboard login | Default user `admin`, default pass `chathura123`. **Change in production.** |
| `JWT_SECRET` | Admin session signing | Auto-generated and persisted to `${DATA_DIR}/.jwt_secret` if not set. |
| `OWNER_NUMBER` | Bot owner WhatsApp number | Used for owner-only commands. |
| `GEMINI_API_KEY` / `OPENAI_API_KEY` / `GROQ_API_KEY` / `OPENROUTER_API_KEY` | AI providers | All optional. |

The dashboard exposes a lightweight `GET /health` endpoint for platform
healthchecks (`/healthz` is also available as a plain-text alias).

### Railway

1. **New Project → Deploy from GitHub** → pick your fork of this repo.
2. Add a **Volume** (Settings → Volumes) and mount it at `/data`.
3. Set env vars: `DATA_DIR=/data`, `ADMIN_PASS=<your_password>`, plus any
   AI keys you want.
4. Railway auto-detects [`railway.json`](./railway.json) and
   [`nixpacks.toml`](./nixpacks.toml) — `ffmpeg` and `yt-dlp` are baked into
   the build, the start command is `node --max-old-space-size=512 index.js`,
   and `/health` is wired up as the healthcheck.
5. Open the generated public URL → `/login` and sign in.

### Render

1. Render → **New → Blueprint** → point at your fork. Render reads
   [`render.yaml`](./render.yaml) and provisions the web service plus a 1 GB
   persistent disk mounted at `/opt/render/project/src/data`.
2. Set the `ADMIN_PASS` env var (marked `sync: false` so you set it
   manually). `JWT_SECRET` is auto-generated by Render.
3. Deploy → open the service URL → `/login`.

### Fly.io / any Docker host

1. Build the included [`Dockerfile`](./Dockerfile) — `ffmpeg` + `yt-dlp`
   baked in, healthcheck wired to `/health`, persistent volume at `/data`.
2. Mount a volume at `/data` and set `DATA_DIR=/data`.
3. Set `ADMIN_PASS` and any AI keys, then run.

### Heroku-style buildpacks

A [`Procfile`](./Procfile) is included (`web: node --max-old-space-size=512
index.js`). Pair the Node.js buildpack with `heroku-community/apt` (or
equivalent) and install `ffmpeg` + `yt-dlp` if you need media downloads.
